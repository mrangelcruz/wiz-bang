## Q&A Log

### Q1 — 17:20
**Question:** Why do we need to delete the state blob before creating a new project?

**Answer:** The existing state blob (`wiz-project-PD.tfstate`) points to the real `GEICO AWS` project by UUID. If Terraform runs an apply with that state still present, it would attempt to modify the live production project rather than create a new one. Deleting the blob forces Terraform to start fresh with no prior state, so the next apply creates a brand-new project instead of updating the existing one.

---

### Q2 — 17:22
**Question:** How do you delete the blob in the Azure Portal?

**Answer:** Navigate to: Storage accounts → `tfstatezscalergznp` → Containers → `tfstate` → locate `wiz-project-PD.tfstate` → select it → click Delete. In the deletion dialog, check "Also delete blob snapshots" to ensure all versions of the blob are removed, then confirm.
